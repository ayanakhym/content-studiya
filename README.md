# Контент Студия — публикация на Vercel (бесплатно, с ссылкой и лёгкими обновлениями)

В этой папке:
- `index.html` — сайт (ключа внутри НЕТ, он берётся с сервера)
- `api/generate.js` — серверная функция-прокси (читает ключ из переменной окружения)

## Публикация (один раз, ~15 минут, без командной строки)

### Шаг 1. GitHub
1. Зарегистрируйся на https://github.com (бесплатно).
2. Нажми «+» вверху справа → **New repository**. Имя любое (напр. `content-studio`). Нажми **Create repository**.
3. На странице репозитория: **Add file → Upload files**.
4. Перетащи сюда **содержимое этой папки**: файл `index.html`, папку `api` (с `generate.js`) и `README.md`.
   Важно: файлы должны лежать в КОРНЕ репозитория (не внутри лишней папки).
5. Внизу нажми **Commit changes**.

### Шаг 2. Vercel
1. Зарегистрируйся на https://vercel.com через кнопку **Continue with GitHub**.
2. **Add New… → Project** → выбери свой репозиторий → **Import**.
3. НЕ нажимай сразу Deploy — сначала открой **Environment Variables** и добавь:
   - Name: `ANTHROPIC_API_KEY`
   - Value: твой ключ Anthropic (формат `sk-ant-...`, берётся на console.anthropic.com → API Keys)
   - Нажми **Add**.
4. Нажми **Deploy**. Через минуту получишь ссылку вида `https://имя.vercel.app` — это твой сайт.

Готово. Ключ хранится в настройках Vercel, в коде и в браузере его нет.

## Как дорабатывать потом
1. Скажи мне, что поменять — я отдам обновлённый `index.html` (или функцию).
2. В GitHub: **Add file → Upload files** → загрузи новый файл поверх старого → **Commit changes**.
3. Vercel увидит изменение и **сам пересоберёт сайт** за минуту. Ссылка та же.

## Важно
- Доступ открытый (без пароля): каждый по ссылке тратит твой баланс Anthropic. Давай ссылку только доверенным. Лимит расходов можно выставить на console.anthropic.com.
- Свой домен можно подключить в Vercel → Settings → Domains (бесплатно).
